| 3회 이상 틀린 후 힌트 보고 정답 | **0원** | 학습은 되지만 적립 없음 ("괜찮아, 다음에 맞추면 돼!") |

| 연속 5문제 정답 | **+2원 보너스** | "5연속 정답! 보너스 2원!" 팝업 |
| 연속 10문제 정답 | **+5원 보너스** | 큰 축하 이펙트 |
| 새로운 단원 첫 도전 | **+3원 보너스** | 새 영역에 도전하는 용기 보상 |

### 3-2. 어뷰징 방지

**문제**: 아이가 이미 마스터한 쉬운 문제만 반복하여 돈을 벌려는 행동

**해결 전략**:

1. **숙달도 기반 적립 체감**  
   - 같은 유형 문제를 10회 이상 연속 정답 시 → 해당 유형의 적립이 1원 → 0.5원 → 0원으로 감소  
   - 화면에 "이건 이제 완벽해! 새로운 도전을 해 볼까?" 메시지  
   - 새로운 난이도로 이동하면 다시 1원 적립 복구

2. **일일 적립 상한**  
   - 하루 최대 적립: **50원**  
   - 상한에 도달하면 "오늘은 충분히 열심히 했어! 내일 또 하자!" 메시지  
   - 부모가 대시보드에서 상한 조정 가능 (30원~100원)

3. **난이도 혼합 출제**  
   - 쉬운 문제만 선택하더라도 자동으로 "도전 문제" 1~2개가 섞여 나옴 (전체의 20%)  
   - 도전 문제 정답 시에는 **2원** 적립

4. **긍정적 넛지 (벌칙 아님)**  
   - 어뷰징을 "막는" 것이 아니라 "새 도전이 더 재미있게" 느끼도록 유도  
   - 새 단원 도전 시 특별 이펙트, 보너스 적립, 부모에게 알림 ("아이가 새 단원에 도전했어요!")

**확장 아키텍처**:

```
[프론트엔드 — 단일 PWA]
```

**핵심 설계 원칙**:
- **오프라인 퍼스트**: IndexedDB에 모든 데이터 저장 → 인터넷 없이도 완전 동작
- **게임 모듈 패턴**: 각 게임이 공통 인터페이스(`init()`, `generateQuestion()`, `checkAnswer()`, `showFeedback()`)를 구현하여 게임 엔진에 플러그인으로 연결
- **문제 데이터 분리**: 문제 풀을 JSON으로 관리하여 코드 수정 없이 확장 가능
- **프레임워크 미사용**: 현재 바닐라 JS 유지 → 빌드 도구 불필요, 배포 단순
